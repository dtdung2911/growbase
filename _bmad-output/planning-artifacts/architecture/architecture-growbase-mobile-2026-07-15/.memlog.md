- (decision) AD-M2 Auth transport: supabase-js(RN) does login+token refresh; access_token sent as Authorization: Bearer to /api. Binds mobile auth + withAuth(). Prevents cookie-only assumption breaking native. BACKEND TOUCH: withAuth() must accept Bearer in addition to cookie
- (decision) AD-M3 Session at rest: LargeSecureStore pattern (AES-256 key in expo-secure-store + encrypted session blob in MMKV) — never raw session in SecureStore (>2048B fails). Biometric (expo-local-authentication) gates app unlock, fallback passcode. Prevents unprotected session + SecureStore size failure
- (decision) AD-M4 Offline model: reads from persisted TanStack Query cache (persistQueryClient+MMKV, gcTime>=maxAge); writes enqueued to durable local mutation queue, replayed in order on reconnect; every mutating /api call carries client-generated Idempotency-Key. Prevents offline data loss + duplicate on retry (NFR-5/SM-C1). BACKEND TOUCH: /api mutation routes honor Idempotency-Key dedupe
- (decision) AD-M5 Shared code: convert to pnpm workspace monorepo; packages/shared holds TS types, Zod schemas, pure business rules, query-key factory keys.* — consumed by web + mobile, no duplication. Prevents web<->mobile drift (NFR-6). Metro needs symlink config per expo-monorepo-example
- (decision) AD-M6 Client conventions mirror web: TanStack Query v5 + Zustand v5; query keys via keys.* (A-4); householdId+currentMonth from Zustand store only (A-7). Single storage layer = MMKV for both query-cache persist and Zustand persist. Prevents divergent state/cache patterns
- (decision) AD-M7 Notification model v1 = LOCAL scheduled only (expo-notifications, {hour,minute,repeats:true} + Android channel). No push server/FCM/APNs in v1. Prevents premature push infra. Advanced push (budget alert, scheduled-due) deferred v2
- (decision) AD-M8 Environments/deploy: Expo EAS Build -> App Store + Play Store. Mobile targets hosted /api via env-configured base URL (dev tunnel/localhost, prod web domain). Native fetch not subject to browser CORS. Binds build+env config
- (decision) AD-M9 Household-scoped cache: persisted MMKV query cache keyed/partitioned by householdId; on household switch PURGE+invalidate persisted cache before loading new. Prevents cross-household data leak (inherited AD-3 contract). Also clear cache on logout
- (decision) AD-M4 AMENDED: offline queue eligibility = only non-balance-sensitive mutations (transaction create/edit/delete). Balance-sensitive ops (fund RPC A-1) are online-only and out of mobile v1 anyway (FR-15). Replay-failure policy: on 4xx conflict surface to user + drop; on 5xx/network retry with backoff.
